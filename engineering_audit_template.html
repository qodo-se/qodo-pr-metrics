Code provenance · Reviewability
Nearly a third of your merged PRs were authored by an AI agent.
Detected via co-author signatures in commit metadata and PR body patterns (Claude, Codex, Cursor, etc.). The category needs more review, not less.

Methodology · how this audit was computed
Source. A single GitHub GraphQL pull, fetched by the qodo-audit.py script you ran. The script reads PRs in the configured org and time window, joins them with reviews, comments, and commit metadata — nothing else. No Qodo product data is read or required.  Window. 60-day lookback ending today. Only merged PRs are included; open and closed-without-merge are filtered out.  "Big" / "fast" thresholds (Section 01). The org's own median lines and hours-to-merge. "Big" = above-median size. "Fast" = at-or-below-median merge time. Computed per-audit, so thresholds reflect your team's normal.  "No human comment" (Sections 01 + 02). Zero non-bot review comments and zero review-level comments on the PR before merge. Bot accounts (dependabot, renovate, your own CI bots) are filtered out before the count.  Final approver (Section 03). The user whose APPROVED review is most recent before merge. PRs merged without an explicit approval are excluded from this section's denominator (135 of 556).  Cycle time (Section 04). PR.mergedAt - PR.createdAt, rounded to the nearest hour.  AI-authored detection (Section 05). PR body co-author signatures (Claude, Copilot, Cursor, Codex) or commit-message patterns matching known agent footprints. Heuristic — not exhaustive. Reported share is a floor.  Re-running. python qodo-audit.py --owner OWNER --since YYYY-MM-DD --until YYYY-MM-DD. Each run produces a fresh standalone HTML file like this one.  Privacy. The script runs entirely in your environment, against your GitHub token. Nothing is sent to Qodo servers. The output HTML can be shared internally, or with us if you'd like a guided walkthrough.
